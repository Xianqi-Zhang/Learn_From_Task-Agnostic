A Memory-Related Multi-Task Method Based on Task-Agnostic Learning

[Under review]

Xianqi Zhang, Xingtao Wang, Xu Liu, Wenrui Wang, Xiaopeng Fan*, Senior Member, IEEE, Debin Zhao, Member, IEEE

VILab, Harbin Institute of Technology

Paper Code

Abstract

     We pose a new question: Can agents learn from task-agnostic data and utilize that knowledge to combine various actions to accomplish new tasks, just like humans? In contrast to imitation learning, there are no expert demonstrations, only the data collected through environmental exploration. Compared with offline reinforcement learning, the distribution shift problem is more serious, because the action sequence to solve a new task may be a combination of trajectory segments of multiple training data, i.e. the solution strategy of the test task does not directly exist in the training set. The challenge is exacerbated by the fact that we do not know what tasks need to be solved in the future during the knowledge-learning. We propose a Memory-related Multi-task Method (M3) to address this problem. The method consists of three stages. First, task-agnostic exploration is carried out to collect data. Different from previous methods, we organize the exploration data into a knowledge graph. We design a model to extract action effect features from the exploration data and save them in memory, while an action predictive model is trained. Secondly, for a new task, the action effect features are used to generate candidate actions by a feature decomposition-based approach. Finally, a multi-scale candidate action pool and the action predictive model are fused to generate a plan to complete the task.


Method Overview

Overview of the Memory-related Multitask Method (M3). Phase 1: Data collection and organization as a knowledge graph with task-agnostic exploration. The features derived by a trained action effect feature extractor are stored in the memory. Phase 2: Employing an action effect feature extractor, we extract task features for specific tasks. The task features are decomposed to generate candidate actions together with the action effect features that are already stored in memory. Phase 3: According to the probability distribution generated by the action predictive model, actions are selected from the multi-scale candidate action pool and executed to generate new states. Continue until the task is finished or the allowable number of steps has been reached. (Optional: Successful plan is executed in the final environment.)
